Site Rules

Control how PhobiaBlocker behaves on specific websites. Add domains or URL patterns to whitelist or blacklist sites.

Whitelist

Websites where PhobiaBlocker will not run at all. No blur will be applied, even if "Always Blur Media" is enabled.

Examples:
  • example.com - Exact domain match
  • *.example.com - All subdomains of example.com
  • example.com/path - Specific path on domain

Blacklist

Websites where blur is always applied to all visual content, regardless of text analysis or other settings.

Note: Blacklist takes precedence over whitelist. If a site is on both lists, it will be blacklisted.

How PhobiaBlocker Works

Overview

PhobiaBlocker is a Chrome extension that blurs visual media on web pages that may contain triggers related to your specific phobias. It uses Natural Language Processing to analyze text around images, videos, and iframes to determine if they should be hidden.

If PhobiaBlocker detects any target words, all visual content on the page will stay blurred to protect you from unwanted content.

How It Works
1. Blur everything on load

2. Run text search with intelligent word extraction

3. Remove blur if nothing is detected

Natural Language Processing (NLP) how it Works?

PhobiaBlocker uses the compromise.js library to analyze text content on the page using AI to search the matches. This allows the extension to understand word variations:

  • Plurals: "cat" matches "cats"
  • Verb forms: "rise" matches "rise", "rose", "rissen"
  • Irregular plurals: "mouse" matches "mice"
  • Possessives: "spider" matches "spider's"
  • Case insensitive: "SPIDER", "Spider", "spider" all match

Flash Prevention & Fail-Safe Design

PhobiaBlocker uses a blur-first, unveil-after-analysis approach to ensure you never see triggering content:

  • CSS-based blur: All visual content is blurred immediately when the page loads, before JavaScript even starts executing
  • Fast analysis: The extension analyzes text content and determines which images are safe
  • Fail-safe default: If anything goes wrong (analysis fails, errors occur, etc.), content stays blurred for your protection

Supported Content Types

PhobiaBlocker can blur:

  • Images: Regular <img> tags, including GIFs
  • Background images: Elements with CSS background-image
  • Videos: <video> elements
  • Embedded content: <iframe> elements (YouTube, Vimeo, etc.)

Dynamic Content Support

PhobiaBlocker uses a MutationObserver to detect when new content is added to the page (infinite scroll, single-page apps, etc.) and automatically analyzes and blurs it as needed. The extension intelligently skips mutations inside text editors to prevent typing lag.

Debug Mode

For troubleshooting only. Enable verbose console logging to diagnose issues.

Debug logging includes:
  • Text Analysis: Word matching, normalization results, phobia word detection
  • Image Detection: Found images, videos, iframes and their blur states
  • Mutation Observer: DOM changes detected and processed
  • Storage: Settings loaded and saved
  • Message Passing: Communication between extension components
  • Editor Detection: Text editors that are skipped to prevent typing lag

To view debug logs, open Developer Tools (F12 or Cmd+Option+I on Mac) and check the Console tab.
